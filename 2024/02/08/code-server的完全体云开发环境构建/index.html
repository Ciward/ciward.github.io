云主机 + code-server的完全体云开发环境构建

​ 最近将电脑重置了一下（接上篇cpu软件问题），本以为又要折腾一天恢复原来的开发环境，结果2个小时就差不多了。自从上次docker安装失败后，我拿出了阿里云白嫖的机子，准备将部分开发环境上云。

系统配置

2核2G，纯净centOS linux 8.4系统

code-server 和 github copilot构建云上生产力工具

​ 出于本人对VScode的极致热爱（毕竟一个软件各种工作），一个不错的计划是部署开源Code-Server，直接在浏览器使用满血版vscode（为什么不用本地加ssh：1.需要本地安装VScode 2.ssh时占内存更大，蒟蒻服务器经常宕机😡😡）。

部署方法

​ Code-Server安装部署全网有众多详细教程，这里主要探讨一下Code-Server的插件问题。（经实测，最好用1.85版本以上的Code-Server，1.80版本插件好像还是无法在线安装，并且github copilot需要重复认证？）

插件安装

  1. 获得微软原生插件市场，具体可参考【WebIDE】 code-server 使用 vscode 原生插件市场地址_code server 换源-CSDN博客。直接向本地一样安装即可
  2. 离线安装：官网下载插件，右侧有个 Download Extension按钮，然后传到服务器，点击从vsix安装即可正常使用。
插件存储位置

​ Code-Server插件和其他用户信息默认存于/root/.local/share/code-server位置，数据迁移时可以直接打包该文件夹。

目前是这样，后续可以补充。
